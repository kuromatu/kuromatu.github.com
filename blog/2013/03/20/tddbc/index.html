TDD Boot Camp Tokyo 2013-03に行ってきた #tddbc

Mar 20th, 2013

  • イベントページはこちら
  • http://tddbc.doorkeeper.jp/events/2904

■会場について

  • 会場はいつもの株式会社VOYAGE GROUP
  • 参加人数は40ぐらい。AJITOではぎゅうぎゅう詰めなので、@t_wadaさんの講演とデモだけAJITO、後はパンゲア（一番広い会議室）で実施
  • Wi-fi環境、飲み物、お酒があるのは楽

■@t_wada さんの講演

  • スピリチュアルな話 + 統計で見る有用性の解説。詳しくはスライドで
  • 「6年くらいデバッガを使ってない」「使うときはIE6と戦うときだけ」
  • 仮実装、三角測量、黄金の回転（出典：スティール・ボール・ラン）重要

■TDDデモ

  • TA（Teaching Assistant）の @yattomさん、 @grimroseさんによるペアプロ。お題はFizzBuzz
  • TDDというよりはペアプロのデモという感じ
  • 作業しながらお互いの考えを尊重しどうコミュニケーションしていくか？という意思がよく伝わってきてとてもわかりやすかった

■TDD実践編

  • monmonさんとペア組み。お題はLTSV。LTSVについてはこちら
  • 自分のPCで機能別に交代で実装していく。準備不足で継続的テスト環境構築（vim + rspec + guard）に手間取ったり、vim、tmuxのキーバインドの違いなどであたふたしてた
  • TODO編集 -> テスト実装 -> RED -> 仮実装 -> GREEN -> 本実装 -> GREEN -> git ci -> 交代 というサイクルが安定してからはかなりハイペースで作業できた

■レビュー

  • ペア相互レビューでは自分達と同じくRuby 2.0 + rspecで開発しているチームで実施。お互いにこういう方向で実装しようとしているよ、ってことを共有した。ここでうまく動いてなかったguardの設定を教えてもらって本当に助かった…
  • 全体レビューではRuby, PHP, Java, C#の代表がそれぞれ発表
  • 開発者別・言語別でのアプローチの違いは大きく、自分たちと対比してみるとそれぞれ利点・欠点が見えて面白い。やっぱり個人的にはRubyがいいなーと思ったり

■懇親会

  • 野良LT合戦。個人的には参加者が持ち寄った「こういう場合、どうしたらいいんですかね」というお悩み相談会的なコンテンツがもっとあっても良かったかも、とあるLTを見て思った
  • マイナー言語（TDDBC的な意味で）の発表は「こうがんばったぜ！」という勢いが見れて面白いですね
  • マサカリ（質問的な意味で）が飛んでくる素敵な会場だと再認識

個人的なまとめ

  • TDDを体験として得られたのが最も大きな成果
  • 各種のおすすめ本を読み直したい。テスト駆動開発入門、レガシーコード改善ガイド、リファクタリングRubyエディションなど
  • pry（デバッガ）を使って試す習慣がついているので、もっとテストとして書き残す習慣にシフトさせたい

■他力本願なまとめ

  • @PoohSunnyさんによるまとめ http://matome.naver.jp/odai/2136365798061279801
